  1. API Document
The system provides APIs to allow easy interact with data in the system.

Authentication

This API will get authenticate token to verify user account.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/api-token-auth/ POST
  • username: username of the account
  • password: password of the account

Keywords

This API allows to query a list of keywords which user monitoring in the system.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/keywords/ GET
  • In the HTTP header, parameter is Authorization: token to authorize the account

Tweets

This API allows to get a list of tweets streaming based on a given keyword.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/tweet_list/?keyword_id= GET
  • keyword_id: id of the keyword which use to stream data

Topics

This API allows to get extracted topics from tweets streaming based on a given keyword.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/topic_list/?keyword_id= GET
  • keyword_id: id of the keyword which use to stream data

Events

This API allows to get extracted events based on a given keyword and a extracted topic.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/event_list/?keyword_id=&topic= GET
  • keyword_id: id of the keyword which use to stream data
  • topic: extracted topic from data

Event knowledge

This API allows to get knowledge extracted from a detected event.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/event_knowledge_list/?keyword_id=&topic=&start_date=&end_date= GET
  • keyword_id: id of the keyword which use to stream data
  • topic: extracted topic from data
  • start_date: start time of event, format of time parameter is "YYYY-mm-dd HH:MM"
  • end_date: end time of event

Enrich knowledge

This API allows to get linking knowledge between extracted knowledge and DBpedia knowledge of an entity.
# URL Method Parameters
1 http://recsys.cau.ac.kr:8086/sociopedia/linking_knowledge/?keyword_id=15&topic=vaccine&start_date=&end_date=&entity= GET
  • keyword_id: id of the keyword which use to stream data
  • topic: extracted topic from data
  • start_date: start time of event, format of time parameter is "YYYY-mm-dd HH:MM"
  • end_date: end time of event
  • entity: extracted entity from event
